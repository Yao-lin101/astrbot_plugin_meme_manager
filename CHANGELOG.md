
## 📜 更新日志

### v4.8.3

- 🧠 **多词/多意图交集精准匹配与评分重构**：
  - 重构了 `match_emotions_by_tags` 函数，支持为每个输入标签召回多达 5 个相似候选词，并在跨标签比对中自动进行去重与顺延。
  - 重构了 `_select_memes_by_emotions_priority` 评分逻辑，对匹配到的意图词采用等比递减结构（第1核心意图固定 1000 分，后续词依次为 400、200、100、50...），在数学上绝对保证除第一个意图外，后面所有意图得分累加之和也无法越过核心意图得分，从而确保了首意图的绝对主导地位。
  - 引入了相似度排位衰减系数（Rank-based Decay），第一候选匹配保留 100% 分值，后面每顺延一位衰减 5%（保底 80%），精细化拉开排序差距，完美解决了由于同分导致随机选到匹配度较低候选词表情包的问题。
  - 限制专属标签额外加分上限为 10 分，仅首个主专属标签参与加分，仅用作微调偏置，不抢占或偏移任何情绪意图。
- 🐛 **缺陷修复与稳定性提升**：
  - 修复了 `load_persona_tags()` 调用未传 `sender.config` 导致在普通消息处理流中人格专属表情标签判定总是静默返回空值失效的 Bug。
- 📋 **日志清理与排版展示优化**：
  - 将频繁输出的标签匹配和向量召回日志（精确匹配标签、候选召回、最终匹配元组列表）降级为 `logger.debug` 层级，清空冗余输出。
  - 重构了“最终发送的表情匹配打分情况”日志，保留为 `logger.info` 级别，输出首行直观展示标签集合，并支持在数据库存在 `description` 时自动显示描述信息，排版更为直观精炼。

### v4.8.2

- 🧠 **独立情绪大模型请求支持 (Emotion LLM)**：
  - 新增独立情绪大模型判定功能 (`enable_emotion_llm`)，允许配置专属的情感分析模型提供商 (`emotion_llm_provider_id`) 和自定义提示词 (`emotion_llm_prompt`)。实现将表情标签提取大模型调用与日常对话回复相解耦，显著提高多标签情绪捕捉的稳定性。
- 🏷️ **专属与偏好标签深度优化**：
  - 自动提取 `meme_use_preference` 里的首个标签作为专属标签，以 70% 概率于匹配完成后追加到最终匹配列表尾部，并自动在情感大模型 Prompt 中屏蔽该标签，避免大模型多余输出。
  - 改进偏好标签解析，支持通过逗号分隔的多标签进行精确与相似度召回。
- 📋 **单标签独立向量召回与原始顺序保持**：
  - 向量召回机制升级为“每个输入标签单独匹配最佳相似度标签”，避免多个不同标签在全局召回中被单一标签主导。
  - 重构匹配结果合并逻辑，使得精确匹配与向量召回结果严格保持 LLM 输出的原始标签顺序，保障下游优先级得分的合理性。
- 🛡️ **发图工具执行旁路拦截 (Tool-Bypass)**：
  - 新增 `meme_tool_executed` 事件上下文标记。若在单次回复中已通过大模型 `send_meme` 原生发图工具成功发送了表情包，将自动拦截并旁路后续 of 被动情绪分析与额外表情匹配，彻底解决双轨模式下重复发送的问题。
- 🐛 **细节修复与日志级别精简**：
  - 修复了因为人设提示词备份污染导致多次热加载后人设文本冗余失效的问题。
  - 将大部分频繁输出的调试日志级别调整为 `logger.debug` 以减少噪音，同时保留关键的向量计算及相似度最佳召回 `logger.info` 日志用于功能验证。

### v4.8.1

- 🏷️ **单通道纯标签向量召回**：
  - 移除了回复文本检索通道与相关的加权配置项（`embedding_tag_weight` 和 `embedding_text_weight`），简化为纯表情标签向量相似度比对，提升向量匹配性能与精准度。
- 📋 **精确与向量匹配顺序保持**：
  - 将匹配和召回结果存储从 set 改为 list 结构，完整保留了大模型输出标签的原始顺序，使下游根据标签位置优先级进行的加权计算与筛选更加合理。
- 🔄 **两步发图工具优化**：
  - 优化了 `send_meme` LLM 工具。支持第二次调用发图时省略 `query` 参数（仅传入 `index`），通过在 `star` 实例上缓存当前会话的候选列表实现两步工作流。发送成功后会自动清理缓存，修复了此前因缺少 query 导致的报错问题。
- 📥 **表情收录/偷图回退取图机制**：
  - 引入了各会话最近收到图片（有效期 5 分钟）的本地缓存机制。在调用 `steal_meme` 工具或自动偷表情时，若当前消息及引用/回复消息中均不含图片，系统将自动回退使用会话最近缓存的图片进行收录，避免漏收或提示无图。
- 👤 **人格偏好读取与代码重构**：
  - 自动偷图判定中的偏好获取直接读取插件在数据库中管理的人格偏好配置（`persona_settings`），不再繁琐地正则解析人格 System Prompt 中的 XML 标签。
  - 将 `steal_meme` 核心处理逻辑移入 `core.llm_tools`，并将大长函数重构拆分为图片下载、相似度去重、偏好审核、分类解析和持久化写入等多段独立子模块，使代码可读性与复用性更高。

### v4.8.0

  - **现有标签参考**：当且仅当“仅分析描述”时，新增“传入现有标签作为分析参考”可选项。勾选后，后端在调用大模型分析前会实时读取该表情包的现有标签，并动态拼接入 Prompt 尾部以供画面和语境参考。

### v4.7.0
